# 其他linux内核相关

vm.overcommit_memory=0|1|2
0 应用程序申请内存时，如果操作系统剩余内存小于申请的值，则直接给应用程序返回错误。
1 内存申请允许超过系统物理内存值，直到使用完成所有物理内存为止，因为内存申请不是申请了就立马使用。
2 决不允许超量申请内存，最大使用内存不能超过 swap +50% RAM（由overcommit_ratio控制）。



vm.dirty_ratio = 10    # 当超过物理内存的10%为脏块，强制做一次磁盘同步，防止缓存中的数据一次性写入磁盘造成卡顿。
